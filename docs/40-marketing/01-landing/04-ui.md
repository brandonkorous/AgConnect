# 01 — Landing Page: UI

The landing page is one long scroll, 18 sections, alternating bone / sage / dark surface bands per [brand/04-spacing-layout.md](../../brand/04-spacing-layout.md). Implementation lives under `apps/web/app/[locale]/(marketing)/`.

**Source of truth for exact dimensions, colors, and copy:** the `Tierra — Landing Page` artboard in Paper. Pull values via `mcp__paper__get_jsx`, `mcp__paper__get_computed_styles`, and `mcp__paper__get_fill_image` — never read from screenshots.

## Routes

| route           | screen                                                                 |
| --------------- | ---------------------------------------------------------------------- |
| `/`             | redirect to `/es` (default locale)                                     |
| `/[locale]`     | landing page (locale ∈ `'en' \| 'es'`)                                 |
| `/[locale]/faq` | standalone FAQ — same content as section 15, plus full FAQPage JSON-LD |

> **Inferred:** `/coming-soon` (referenced by Phase A CTAs before auth ships) is NOT part of this feature folder; it lives in a sibling `40-marketing/05-coming-soon` folder. Out of scope for MVP if auth ships in Phase 0 alongside Phase A — in that case all CTAs route to real flows from Day 1.

## Container and grid

- Outer page: bone (`#EFE6D2`).
- Each section is a full-width band with internal `max-width: 1280px` content and `padding-x: 80px` desktop / `32px` tablet / `20px` mobile.
- Vertical padding between major sections: `96px` (`space-24`) desktop, `80px` tablet, `64px` mobile.
- Grid: 12 col / 24px gutter desktop; 8 col / 20px tablet; 4 col / 16px mobile.

## Surface band rhythm

Per [brand/04-spacing-layout.md](../../brand/04-spacing-layout.md), landing alternates bands. Locked rhythm:

| section                | band                                   | rationale                                                        |
| ---------------------- | -------------------------------------- | ---------------------------------------------------------------- |
| Utility Bar            | bone                                   | top of page neutral                                              |
| Nav                    | bone                                   | continuous with utility bar                                      |
| Hero                   | bone                                   | spaciousness; phone mockup pops                                  |
| Trust Strip            | bone                                   | continuous                                                       |
| Audience Split         | bone (with internal moss + sage cards) | section header on bone, cards bring color                        |
| Worker How It Works    | sage                                   | sectional shift; 4 steps feel set apart                          |
| Employer Showcase      | bone                                   | back to default; dashboard mockup is the focal element           |
| Verification Spotlight | bone                                   | bilingual neighbor of showcase                                   |
| Bilingual Section      | sage                                   | brand narrative beat                                             |
| Impact Numbers         | moss (dark band #1)                    | the "deep stop" — numbers feel weighty on dark                   |
| Featured Jobs          | bone                                   | back to neutral for content browse                               |
| Featured Training      | bone                                   | continuous                                                       |
| Testimonials           | sage                                   | quotes feel set apart                                            |
| Pricing                | moss (dark band #2)                    | pricing on dark — the brand's only loud color (honey) shows here |
| FAQ                    | bone                                   | back to neutral for reading                                      |
| Final CTA              | bone                                   | the conversion moment, calm                                      |
| Footer Main            | ink (dark band #3, exception)          | footer convention                                                |
| Footer Legal           | ink                                    | continuous                                                       |

> **Inferred:** Three dark bands violates "at most twice per page" from the brand spacing doc. The footer is conventionally dark and read as an off-page region; counting it as one of the two would force Pricing or Impact Numbers off dark, weakening both. Decision: footer is exempt from the dark-band cap. If a designer disagrees, lift Impact Numbers to bone-with-honey-accents and keep Pricing dark.

## States

- **Loading** — RSC; static content paints first. Featured Jobs / Training / Impact use streaming via `<Suspense>` boundaries with skeleton fallbacks (matching final card shape, no layout shift).
- **Error** — if `/v1/landing/*` fails, fall back to mock data with a small in-page banner only visible to admins (logged in admin role sees a "data unavailable" hint).
- **No data** — Featured Jobs empty? Hide section. Featured Training empty? Hide section. Impact suppressed (count < 25)? Show "Coming soon" copy, keep tile shape.
- **Anonymous vs authenticated** — anonymous sees CTAs to sign-up flows; authenticated worker sees `Open dashboard` CTA in nav and a small "Welcome back, María" line in the hero subhead area (still showing the same hero — landing remains useful as a public reference even when signed in).

## Accessibility

Per [brand/08-accessibility.md](../../brand/08-accessibility.md):

- Visible focus: 2px outline (honey on bone, bone on dark) — no shadow.
- All interactive controls keyboard-reachable in source order.
- `<html lang>` matches the active locale.
- Alt text on every image in both locales.
- FAQ accordion keyboard: Space/Enter toggles, Up/Down navigates between questions.
- Skip-to-main-content link visible on focus.
- Color contrast verified per [brand/02-color.md](../../brand/02-color.md) pairings.
- All hero CTAs have ≥ 44×44 touch targets.

## Mobile

- Single-column layout below 768px.
- Hero phone mockup: shrinks to 280×388, stays right-aligned but smaller; toast moves below the phone.
- Audience Split: cards stack; sticky tab strip lets users jump between Workers / Employers / Training Orgs.
- Featured Jobs / Training: horizontal scroll-snap on mobile, 4-up grid on desktop.
- Pricing: cards stack vertically; "Most popular" Pro card highlighted with honey ribbon.
- FAQ: same accordion (no layout change).
- Final CTA: cards stack; both forms full-width.

## Performance

- All marketing pages are RSC + ISR (`revalidate: 300` on the page; per-fetch revalidate set in [03-api.md](03-api.md)).
- LCP target on hero: < 2.0s on mobile 4G. Optimization:
    - Server-render the hero entirely (no client component above the fold).
    - Hero phone mockup is HTML/CSS, not an image.
    - `next/font` for Fraunces / Inter / DM Mono with `display: 'swap'`.
    - Preload the Fraunces variable italic file (the brand-defining cut).
- Total client-side JS budget on landing: < 50 KB gzipped (most of the page is RSC).
- Use `next/image` with `priority` on the hero phone wrapper (if any image fills exist after Paper export).
