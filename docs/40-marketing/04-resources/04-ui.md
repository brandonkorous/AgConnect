# 04 — Resources: UI

## v1 launch posture

`/[locale]/resources` exists as an empty-state page until v2 lands real content.

### Empty-state layout

```
┌─────────────────────────────────────┐
│  [Utility Bar]  (from layout)       │
│  [Marketing Nav]                    │
├─────────────────────────────────────┤
│                                     │
│     Eyebrow: RESOURCES              │
│     Headline: Long-form guides,     │
│               coming soon.          │
│     Body: Articles on farmworker   │
│           rights, FLC compliance,   │
│           and training programs are │
│           rolling out shortly.      │
│           Sign up below to get them │
│           in your inbox.            │
│                                     │
│     [Email input]   [Subscribe →]   │
│                                     │
│     Want to suggest a topic?        │
│     Email editorial@agconn.com →    │
│                                     │
├─────────────────────────────────────┤
│  [Marketing Footer]                 │
└─────────────────────────────────────┘
```

Behavior:

- Inherits `(marketing)/layout.tsx` for chrome.
- Subscribe form posts to `POST /v1/landing/waitlist` with `source: 'landing.resources_waitlist'` (a new enum value added to the existing waitlist endpoint).
- Confirmation: same email-confirmation flow as the existing landing waitlist.
- No cards, no list — page is intentionally sparse to make the "coming soon" message clear and avoid suggesting articles exist.

> **Inferred:** Reusing the waitlist endpoint avoids spinning up a `subscriptions` table for v1. The `source` field already discriminates between landing-final-CTA and resources-coming-soon, so reporting can split them later.

### Link visibility in nav

The "Resources" nav link is visible (per landing's `<MarketingNav>`) and routes to `/[locale]/resources`. Hiding the link feels worse than showing the empty state — visitors who want articles know they're coming.

Layout:

```
┌─────────────────────────────────────┐
│  Eyebrow: RESOURCES                 │
│  Headline: Guides for the field.    │
│  Body: Plain-language articles for  │
│        workers, employers, and      │
│        training organizations.      │
│                                     │
│  [All] [Workers] [Employers]        │
│  [Training] [Platform]              │
│                                     │
│  ┌─────────┐  ┌─────────┐           │
│  │ Article │  │ Article │           │
│  │ card 1  │  │ card 2  │           │
│  └─────────┘  └─────────┘           │
│  ┌─────────┐  ┌─────────┐           │
│  │ Article │  │ Article │           │
│  │ card 3  │  │ card 4  │           │
```

- Two-column grid desktop (`grid-cols-2 gap-8`), single column mobile.
- Each card: hero image (16:9), category eyebrow, Fraunces 24px title, Inter 15px description, reading time + date footer.
- Filter chips at top — daisyUI tabs styling, Tierra theme.
- Sort: most recent first (by `publishedAt`).

```
│  │ Sec 2    │  Sec 2 …          │   │

│                                     │

└─────────────────────────────────────┘
```

- Two-column desktop: 220px sticky ToC left, 760px article body right.
- Single column mobile; ToC collapses into a `<details>` summary at top.
- MDX components: `<H1>`, `<H2>`, `<H3>` (Fraunces), `<p>` (Inter 17px on `760px` reading width), `<ul>` / `<ol>`, `<blockquote>` (sage background, soil left border), `<Pull>` (custom — Fraunces italic 28px pull-quote).
- Code blocks: rendered via `shiki` with Tierra-themed syntax highlighting.
- Inline images: `next/image` with `sizes` attribute.
- Footnotes: standard MDX numbered footnotes.

### Category index `/[locale]/resources/category/[category]`

Same layout as the main index, filtered to that category, with the category name in the page header.

## Accessibility

- Article body: max-width `760px` for ideal reading line length (~75 chars).
- Headings semantic and ordered (no skipped levels).
- Skip-link to article body.
- ToC links use `aria-current="location"` on the active section.
- Hero images have alt text in both locales.

## Mobile

- Single column.
- Sticky ToC becomes a top `<details>` block; collapsed by default.
- Article body padding: 20px sides.
- Hero image: full-bleed.
- Related articles: horizontal scroll-snap.

## Performance

- Articles are fully RSC.
- Client JS budget per article: < 20 KB gzipped (just ToC scroll-spy).
- LCP target on article: < 2.0s on mobile 4G — hero image gets `priority` + responsive sizes.
- ISR `revalidate: 3600` per article (1 hour); manifest revalidate hourly.
