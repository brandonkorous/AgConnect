# 02 — FAQ: UI

## Route

`/[locale]/faq` — sits inside the `(marketing)` route group, so it inherits the `<UtilityBar>` + `<MarketingNav>` + `<MarketingFooter>` chrome from `(marketing)/layout.tsx`.

## Layout

Full-bleed page (matches landing's `container mx-auto lg:px-20`). Two-column at `lg+`: left column carries the eyebrow + Fraunces headline + intro + a small "have-a-question-we-missed" card, sticky-positioned; right column carries the accordion. Below `lg`, columns stack and the accordion renders full-width inside the container.

The `max-width: 760px` callouts in earlier drafts of this spec were a **line-length cap for prose** within the right column (so paragraphs don't run past the 65–75 character ideal), NOT a page-width cap. The page itself bleeds full-width per [brand/04-spacing-layout.md](../../brand/04-spacing-layout.md).

96px vertical padding desktop / 64px mobile. Bone background.

## Page header

- Eyebrow: `FAQ` — uppercase, 0.18em tracking, soil color, Inter 13px semibold (uses `<EyebrowLabel>`)
- Headline: `Questions, in plain language.` — Fraunces 64px desktop / 48px mobile, ink, semibold; same i18n key as landing FAQ section (`landing.faq.headline`)
- Intro: Inter 17px, ink @ 70%, 60ch max — `marketing.faq_page.intro` ("Common questions about how AGCONN works for workers, employers, and training organizations.")

## FaqAccordion (reused component)

Already exists at [apps/web/src/components/landing/FaqAccordion.tsx](apps/web/src/components/landing/FaqAccordion.tsx). Props extension needed:

```ts
interface FaqAccordionProps {
  entries: FaqEntry[];
  initialOpen?: string[];        // ids open by default; empty array on /faq, ['how-find-work', 'flc-verification'] on landing
  variant?: 'embedded' | 'standalone'; // standalone: full-width within container, no card chrome
}
```

Behavior:

- Each item: hairline top border, Fraunces 20px question, Inter 16px answer, `+`/`−` (FontAwesome `faPlus`/`faMinus`) toggle, 24px vertical padding.
- Click toggles. Multiple can be open at once.
- Keyboard: Tab to move between questions (each is a `<button>` inside an `<h3>`); Space/Enter toggles; Up/Down arrow keys move between question buttons (custom keydown handler).
- Anchor support: on mount, parse `window.location.hash`. If it matches an entry id, mark that one open and `scrollIntoView({ behavior: 'smooth', block: 'start' })` after layout.

> **Inferred:** Anchor handling runs client-side — the SSR shell renders all closed, then a small client effect opens the targeted entry. Avoids hydration mismatches.

## Extras separator

Between the 8 shared entries and the extras: a hairline divider plus a small caption:

- Caption: `More detail` (EN) / `Más detalle` (ES) — Inter 13px, soil, uppercase 0.14em tracking.

This signals "you've reached the end of the landing FAQ; below is more granular".

## "Still have questions?" CTA

Bottom of page, after the accordion:

- Bone surface card, hairline border, 32px padding, max-width same as content
- Headline: Fraunces 24px italic — `Still have questions?` / `¿Más preguntas?`
- Body: Inter 15px — short paragraph
- Action: `Email support@agconn.com →` (mailto link, moss text, hover underline)
- Secondary: `Back to home →` (Inter 14px, soil)

## States

- **Loading** — RSC; the page is fully static. No client loading state.
- **No-JS** — accordion items render with all answers visible (default `<details open>` semantics fallback). The `+`/`−` controls become CSS-only via `<details>` for graceful degradation.

> **Inferred:** Build the accordion as `<details><summary>…</summary>…</details>` so it works without JavaScript by default. The custom keyboard arrow handler is JS-enhanced; without JS, native browser `<details>` keyboard behavior takes over (Space/Enter toggle, Tab navigation).

## Accessibility

- `<h1>` for the page headline; `<h3>` for each question (semantic outline).
- `<details>` element provides screen-reader announcements for open/closed.
- Visible focus on every question button: 2px honey outline on bone.
- Color contrast: question 7:1 on bone (ink), answer 4.5:1 (ink @ 70% on bone).
- All-collapsed default per [01-overview.md](01-overview.md#structure-of-faq) — first interaction is user-initiated.

## Mobile

- Single column already; minimal layout change.
- Fraunces headline scales to 48px.
- Question font scales to 18px; answer stays 16px.
- 16px page padding sides.

## Performance

- Page is fully static; no client JS for content.
- Client JS budget: < 5 KB gzipped (just the accordion arrow-key keydown handler).
- LCP target: < 1.5s on mobile 4G — the page is text-only, no images.
