## Public verified badge

On employer profile pages and job cards. The displayed name is `dbaName ?? legalName`:

```
Central Valley Farms ✓
```

Hover/tap → tooltip shows two lines so workers see both names:

```
AgConn verified employer
Legal entity: Central Valley Agriculture, LLC
```

If `dbaName` is null (legal name is the public name), the second line is omitted.

## States & behavior

- **Not verified employer hits POST /publish** → API 403; UI shows "You're not verified yet" modal with "Edit info" link.
- **Verified employer downgrades to pending** (license # changes) → existing `active` jobs stay active (employer's word matters here; doesn't auto-unpublish), but new publish blocked. Admin re-reviews.
- **Verification expires** (Phase 2 scraper) → flips to `flcVerifiedAt = null`; same behavior as never-verified (new publish blocked, existing stay).

> **Inferred:** Don't auto-close existing jobs on verification expiry — that's punitive and may hurt workers actively applying. Block new publishes only. Admin can manually close jobs if a verification fails egregiously.
