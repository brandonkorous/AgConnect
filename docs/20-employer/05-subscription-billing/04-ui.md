# 05 — Subscription Billing: UI

## Routes

| route | screen |
|---|---|
| `/[locale]/employer/billing` | current plan + manage |
| `/[locale]/employer/billing/plans` | plan comparison + upgrade |
| `/[locale]/employer/billing/success` | post-checkout landing |
| `/[locale]/pricing` | public pricing page (marketing) |

The marketing pricing page is also the entry point for the founder badge. Both surfaces consume `GET /v1/landing/founder-slots`.

Auth-gated `/employer/billing/*` routes require `userRole === 'employer'`.

## Public pricing page (marketing)

`/[locale]/pricing` is the public version of the comparison: same three cards (Seed / Field / Farm), same founder-spots badge, plus supporting bands for "Workers — always free" and "Training organizations — always free." The page is RSC + ISR (`revalidate = 86400`); the founder-spots badge is fetched client-side or via a 30s-cached server fetch so the page can stay statically rendered.

CTA targets:

| card | CTA | destination |
|---|---|---|
| Seed | Start free | `/[locale]/employer/sign-up` |
| Field | Start at founder pricing / Get started | `/[locale]/employer/sign-up?plan=field` |
| Farm | Talk to sales | `mailto:sales@agconn.com?subject=Farm%20plan` |

Field and Farm CTA labels swap based on `founder.active`:

- `founder.active === true` → "Start at founder pricing" / "Get started"
- `founder.active === false` → "Get started" / "Get started"

## Checkout success

After Stripe Checkout, employer redirected to `/billing/success?session_id=...`. Page polls `/v1/employer/billing` until `plan != 'free'` (webhooks may take seconds), then shows:

```
┌────────────────────────────────────┐
│  ✓ You're on Field!                │
│                                    │
│  All Field features are now active.│
│                                    │
│  [Go to dashboard]                 │
└────────────────────────────────────┘
```

If the cohort was founder, sub-line: "Locked at founder pricing — your renewal stays at $99/mo as long as you keep your subscription."

If polling > 30s without update, show "Your subscription is processing. We'll email you when it's active." and stop polling.

## Plan downgrade

In Stripe Customer Portal — handled by Stripe's UI. We sync via webhook.

If employer has > 2 active postings on Field and downgrades to Seed:

- Stripe webhook updates `plan = 'free'`.
- Existing 5 postings remain active until their end date.
- New publish blocked at 2-active limit.

## Failed payment banner

If `subscription.status = 'past_due'` (Stripe Smart Retries in progress), show banner across employer dashboard:

```
⚠ Your last payment failed. Update your card to avoid losing Field features.
[Update payment]
```

After 3 failed retry attempts (Stripe's default), Stripe cancels the subscription → webhook → plan = free. The freed slot returns to the founder pool.

## Components

- `<BillingHome>`, `<PlansComparison>`, `<PlanCard>`, `<FounderBadge>`, `<CheckoutSuccessPolling>`, `<PaymentFailedBanner>`, `<PlanGate>` — in `apps/web/src/app/[locale]/employer/billing/_components/`.
- Public marketing equivalents — `<Pricing>`, `<PlanCard>`, `<FounderBadge>` — in `apps/web/src/components/landing/`.
- The marketing pricing page (`apps/web/src/app/[locale]/(marketing)/pricing/page.tsx`) renders three Seed/Field/Farm cards, plus auxiliary "always free" bands for workers and training orgs.

## Performance

- Public pricing page is RSC + ISR; founder-spots badge fetched separately and revalidated every 30s on the edge.
- Billing home RSC; calls `/v1/employer/billing` server-side.
- Stripe Checkout / Portal hosted by Stripe — no client-side rendering.
