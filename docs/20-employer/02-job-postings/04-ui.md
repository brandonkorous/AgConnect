# 02 — Job Postings: UI

## Routes

| route | screen |
|---|---|
| `/[locale]/employer/dashboard` | employer home (incl. recent postings) |
| `/[locale]/employer/jobs` | full list of own postings |
| `/[locale]/employer/jobs/new` | create form |
| `/[locale]/employer/jobs/:id` | edit + applicant tab links |
| `/[locale]/employer/jobs/:id/preview` | preview as worker |

Auth-gated to `userRole === 'employer'`.

### Translate button

Tap → modal: "Translate from English (or Spanish)?"  → API call → result populates the other-language field with green-pulsing border to draw the user's eye for review.

> **Inferred:** Visible "Review this" prompt on freshly-translated text. The employer's review pass is the safeguard for translation quality.

### Validation

- Real-time on blur: red error below field.
- Submit-time: full Zod check; first error scrolls into view.
- Publish button disabled until all required fields valid.

## Preview as worker

Renders the posting using the worker-side detail page component. Lets the employer see exactly what workers will see.

## Status badges

`<JobStatusBadge>` (in `packages/ui`):

```
draft   = gray "Draft"
active  = green "Live"
filled  = blue "Filled"
closed  = gray "Closed"
```

## Confirmation modals

- Publish: confirm "Make this posting visible to workers?"
- Close: confirm "Close this posting? Workers will no longer be able to apply."
- Delete draft: confirm "Delete this draft? This can't be undone."

## States

- **Empty list:** "You haven't created any job postings yet. [+ New posting]"
- **Plan limit hit:** posting card shows "Reached your free plan limit. [Upgrade]"
- **Edit forbidden field on active:** inline message "This field can't be changed while the posting is active. Close and create a new posting to change [field]."
- **Translation failure:** banner "Translation failed. Please enter text manually."

## Components

- `<EmployerJobList>`, `<JobForm>`, `<TranslateButton>`, `<JobStatusBadge>`, `<JobPreview>`, `<PlanLimitBanner>`, `<SkillChipPicker>` — in `apps/web/app/[locale]/employer/jobs/_components/`.

## Performance

- Form is client-rendered (highly interactive).
- Autosave debounced 800ms (longer than profile because forms are large).
- Translation calls are async; show spinner per field, never block the whole form.
