# 04 — Worker Search: UI

## Routes

| route                            | screen                                    |
| -------------------------------- | ----------------------------------------- |
| `/[locale]/employer/workers`     | search results                            |
| `/[locale]/employer/workers/:id` | worker preview                            |
| `/[locale]/me/invitations`       | worker's invitation inbox                 |
| `/[locale]/invitations/:id`      | worker invitation detail (accept/decline) |

## States

- **No results:** "No workers match these filters. Try removing one." with quick-clear chip per active filter.
- **Already invited:** worker preview shows banner "You invited Maria on Apr 10".
- **Already applied:** banner "Maria already applied to this posting" + link to applicant detail.

## Components

- `<WorkerSearchInterface>`, `<WorkerCard>`, `<WorkerPreview>`, `<InviteModal>`, `<PlanGate>`, `<InvitationDetailPage>` — in `apps/web/app/[locale]/employer/workers/_components/`.

## Performance

- Search RSC + client island for filters.
- Filter changes use `router.replace(?...)` so URLs are shareable.
- Pagination via cursor.

## Privacy in UI

- All worker cards / previews explicitly show `firstName + lastInitial` — no full name component used.
- Phone / email rendering throws an error if attempted — defensive coding.
