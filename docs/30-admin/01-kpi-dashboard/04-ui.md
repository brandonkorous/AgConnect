# 01 — KPI Dashboard: UI

Lives in `apps/admin`, Next.js 16. English-only.

## Routes

| route                | screen                                     |
| -------------------- | ------------------------------------------ |
| `/`                  | dashboard home (this feature)              |
| `/placements`        | placement report drill-down                |
| `/training`          | training report drill-down                 |
| `/employers`         | employer activity drill-down               |
| `/tenants`           | (admin tenant CRUD; multi-tenancy feature) |
| `/employers/pending` | (FLC verification queue)                   |

## Tile component

```tsx
<KpiTile
    title='Hires'
    value={142}
    trend={{ deltaPct: 12, deltaAbs: 15 }}
    series={
        [
            /* sparkline data */
        ]
    }
    onClick={() => router.push('/placements')}
/>
```

Trend arrow + color: green up if positive, red down if negative. Magnitude in parentheses.

## Filters

Filter bar at top, sticky on scroll:

- **Tenant**: select for AgConn-wide admins; locked for tenant-scoped admins (Phase 2).
- **County**: All / multi-select chips.
- **Range**: presets (This week / month / quarter) + custom date picker.

Filter changes update URL query string for shareability:

```
/?tenant=00000000-0000-0000-0000-000000000001&county=Fresno&start=2026-01-01&end=2026-04-01
```

## Auto-refresh

Optional toggle "Auto-refresh every 60s". Off by default. When on, polls `/admin/v1/kpi/summary` every 60s. Last-updated timestamp visible.

## Export

"Export" button → CSV of the currently-displayed tile values + breakdowns. Also available from the drill-down reports.

## Drill-down

Tap any tile → routes to the corresponding report (`/placements`, `/training`, `/employers`) with the same filters applied (filters carry via URL params).

## Loading state

Tiles render skeletons while loading. If a tile errors, it shows a small "—" with a retry icon (other tiles render independently).

## Empty state

Brand-new tenant with no data: dashboard shows tiles with `0` values and a banner: "No data yet — start by inviting employers."

## Components

- `<KpiTile>`, `<KpiFilters>`, `<KpiBreakdown>`, `<WageHistogram>`, `<RefreshButton>`, `<ExportButton>` — in `apps/admin/app/_components/kpi/`.

## Performance

- RSC initial render with cached `/v1/kpi/summary` response.
- Filter changes use client-side fetch with SWR (stale-while-revalidate) for snappy UX.
- Charts use lightweight library (e.g., `react-chartjs-2` or hand-rolled SVG sparklines for tiles).
