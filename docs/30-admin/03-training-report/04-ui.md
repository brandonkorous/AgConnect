# 03 — Training Report: UI

Admin app. English-only.

## Routes

| route | screen |
|---|---|
| `/training` | report builder |
| `/reports/runs` | shared with placement-report; filtered by `reportType = 'training'` |

## Report builder

Mirrors `/placements`:

```
┌─────────────────────────────────────┐
│  Training Report                    │
│                                     │
│  View:                              │
│  ( ) Per enrollment                 │
│  ( ) By program                     │
│  ( ) By funder                      │
│  ( ) By training org                │
│                                     │
│  Filters                            │
│  Tenant: [All ▾]                    │
│  Funder: [All ▾] [CDFA ✓] [F3 ✓]   │
│  Training org: [All ▾]              │
│  County: [All ▾]                    │
│  Date range: [Start] – [End]        │
│  Date basis: ( ) Enrollment date    │
│              ( ) Completion date    │
│  ☐ Include worker names             │
│                                     │
│  Format: ( ) CSV  ( ) XLSX           │
│  Delivery: ( ) Download ( ) Email   │
│                                     │
│  [Preview]  [Generate]              │
│                                     │
│  Preview                            │
│  [scrollable table]                 │
└─────────────────────────────────────┘
```

The `view` selector toggles between row-level and aggregate views; preview table layout adapts.

## States

Same as placement-report. Empty preview, generation progress, email confirmation.

## Components

- Reuse `<DateRangePicker>`, `<FilterChips>`, `<ReportPreviewTable>` from placement-report.
- New: `<TrainingReportBuilder>`, `<ViewSelector>` — in `apps/admin/app/training/_components/`.
