Timeline rendered from `application_events`. Withdrawn / hired / rejected end-states close the timeline; "Withdraw" button hidden for those.

If hired: show start date + employer contact info ("Central Valley Farms will contact you. You can also reach them at <phone>.").

## States

- **Loading:** skeleton matching final layout.
- **Empty active tab:** "You haven't applied to any jobs yet. [Browse jobs]"
- **Empty hired tab:** "Once you're hired for a job, it'll appear here."
- **Withdraw confirmation modal:** "Withdraw your application for {job}? This can't be undone." вҖ” Cancel / Withdraw.

## Status badges

Reusable `<StatusBadge>` component (in `packages/ui`) with i18n labels and colors:

```tsx
const STATUS_VARIANTS: Record<AppStatus, { color: string; key: TranslationKey }> = {
  applied:   { color: 'bg-info text-info-content', key: 'application.status.applied' },
  reviewed:  { color: 'bg-warning text-warning-content', key: 'application.status.reviewed' },
  hired:     { color: 'bg-success text-success-content', key: 'application.status.hired' },
  rejected:  { color: 'bg-base-300 text-base-content', key: 'application.status.rejected' },
  withdrawn: { color: 'bg-base-300 text-base-content', key: 'application.status.withdrawn' },
};
```

## Notifications

The dashboard navigation includes a small badge counter showing applications with unread status changes. "Unread" tracked client-side via localStorage for MVP (server-side notification table out of scope).

## Performance

- All application list/detail pages are RSC.
- Timeline events fetched in the same RSC pass as the application detail.
- ISR with low revalidate (60s) on the list; detail revalidates on every visit (no ISR).
