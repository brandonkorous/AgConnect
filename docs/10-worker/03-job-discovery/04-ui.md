# 03 — Job Discovery: UI Flow

## Routes

| route | screen |
|---|---|
| `/[locale]/jobs` (or `/es/trabajos`) | listing + filters |
| `/[locale]/jobs/[slug]` | job detail |
| `/[locale]/saved-searches` | manage saved searches (auth) |

## Listing screen

Layout (mobile-first):

```
┌────────────────────────────────┐
│  Jobs in Central Valley        │
│  [Search bar]                  │
│  [Chip: Fresno] [Chip: Tractor]│  ← active filters
│  [Filters ▼] [Save search]     │
│                                │
│  ┌────────────────────────────┐│
│  │ Strawberry Picker          ││
│  │ Fresno • $18-22/hr • Apr 20││
│  │ Central Valley Farms ✓     ││
│  └────────────────────────────┘│
│  ┌────────────────────────────┐│
│  │ ...                        ││
│  └────────────────────────────┘│
│  [Load more]                   │
└────────────────────────────────┘
```

- Each job card is a single tap-target leading to `/jobs/[slug]`.
- The card shows: bilingual title (active locale), county, wage range, start date, employer name + verified badge.
- Search bar uses a debounced (250ms) request to `/v1/jobs?q=...`.
- "Save search" CTA persistent at top-right of filters; tap → modal asks for name + alert opt-in.

### Filter sheet

Bottom-sheet modal on mobile, dropdown on desktop:

- County (multi-check, 5 options)
- Skill (multi-select chips, default tags + custom)
- Wage range (slider $0–$50/hr)
- Start date (`Before <date>` / `After <date>`)
- Employer (search input)
- "Apply filters" + "Clear all" CTAs

### Empty / zero-result states

- No filters, no matches: "No jobs in your area right now. We'll text you when new ones appear if you save this search." + Save Search CTA.
- With filters: "No jobs match these filters. [Clear filters]"

### Loading

- Skeleton cards (5) match final card layout.
- After 30s loading without response: "Still loading — your connection might be slow."

## Job detail screen

```
┌────────────────────────────────┐
│  ← Back to jobs                │
│  Strawberry Picker             │
│  Cosechador de Fresas          │
│                                │
│  Fresno County • Apr 20 - Jun 10│
│  $18 - $22 / hour               │
│                                │
│  Central Valley Farms ✓        │
│  View employer profile →        │
│                                │
│  [Apply now] (large CTA)       │
│                                │
│  About the job                 │
│  ────────────────────────       │
│  <description in active locale>│
│                                │
│  Skills needed                 │
│  [Harvesting] [Pruning]        │
│                                │
│  Share this job  [WhatsApp ▼] │
└────────────────────────────────┘
```

- Apply CTA is the largest interactive element; tap → goes to apply confirmation (see application-tracker).
- For unauthenticated users: Apply CTA reads "Sign in to apply" → routes to `/sign-in?redirect=/jobs/<slug>/apply`.
- Employer link → public employer profile page (verified employers).
- Share dropdown: WhatsApp (web.whatsapp.com), SMS (`sms:?&body=...`), copy link.
- Above-the-fold: title + key facts. Description below the fold; skills further below.

### SEO additions

- `JobPosting` JSON-LD rendered server-side (see [00-foundation/09-seo-aio/02-data-model.md](../../00-foundation/09-seo-aio/02-data-model.md)).
- Direct-answer paragraph at the top of description ("This is a seasonal harvesting position in Fresno County. The wage is $18–22/hour. Applications open through April 18.").

## Saved searches screen

Authenticated workers only. Shows:

```
┌────────────────────────────────┐
│  Your saved searches           │
│  ┌────────────────────────────┐│
│  │ Fresno + Tractor           ││
│  │ SMS alerts: ON [toggle]    ││
│  │ [Edit] [Delete]            ││
│  └────────────────────────────┘│
└────────────────────────────────┘
```

Empty state: "You haven't saved any searches yet. Save a search from the jobs page to get SMS alerts when new jobs match."

## Recommendations on dashboard

`apps/web/app/[locale]/dashboard/page.tsx` (post-onboarding home):

- Top section: "Recommended for you" — up to 5 cards from `/v1/jobs/recommended`.
- Below: saved-search shortcuts.
- Below: "All jobs in your area" linking to `/jobs?county=<my-county>`.

## States

- **Anonymous** can browse, search, view detail, but cannot apply or save.
- **Authenticated worker without onboarding complete** is redirected to onboarding (no jobs UI).
- **Authenticated worker** sees apply, save, recommendations.

## Performance

- Listing page is RSC (server-rendered) for SEO + low TTFB.
- Filters update via client navigation (`router.push` with new query params).
- Detail page is RSC; share button is client component.
- ISR with 60s revalidate on the listing page (job updates take up to 60s to propagate to crawlers; for authenticated views, fetch fresh data client-side).

## Components

- `<JobCard>`, `<JobDetail>`, `<JobFilters>`, `<JobShareMenu>`, `<SavedSearchCard>`, `<SaveSearchModal>` — in `apps/web/app/[locale]/jobs/_components/`.
