# 01 — Worker Onboarding: UI Flow

## Routes

`[locale]` is `en` or `es`. `next-intl` middleware redirects `/` → `/es` by default for first visits with no preference.

| route                                 | screen                                      |
| ------------------------------------- | ------------------------------------------- |
| `/[locale]/welcome`                   | landing + language toggle                   |
| `/[locale]/sign-in`                   | phone entry                                 |
| `/[locale]/sign-in/verify`            | OTP entry                                   |
| `/[locale]/onboarding/language`       | confirm preferred language                  |
| `/[locale]/onboarding/resume`         | resume upload OR skip                       |
| `/[locale]/onboarding/resume/parsing` | spinner with progress hint                  |
| `/[locale]/onboarding/profile`        | parsed-data review/edit                     |
| `/[locale]/onboarding/county`         | county picker                               |
| `/[locale]/onboarding/skills`         | skill multi-select                          |
| `/[locale]/onboarding/availability`   | weekly availability grid                    |
| `/[locale]/onboarding/complete`       | success + first job recommendations         |
| `/[locale]/onboarding/waitlist`       | shown when user picks an unsupported county |

The middleware in `apps/web/middleware.ts` redirects authenticated workers with `onboardedAt = null` to the next incomplete step on every page load.

## Step-by-step flow

### 1. Welcome (`/[locale]/welcome`)

- Big language toggle: "English" / "Español" — full-width buttons, large tap targets (≥ 60px height).
- Tagline in both languages above the toggle.
- "Start" CTA below.
- Single page; no scroll on mobile.

### 2. Phone entry (`/[locale]/sign-in`)

- Single input: 10-digit US phone, formatted as `(555) 555-5555` via `inputmode="tel"` and a mask.
- Helper text: "We'll text you a 6-digit code. Standard message rates apply."
- Submit calls Clerk's `signIn.create({ strategy: 'phone_code', identifier })`.

### 3. OTP verify (`/[locale]/sign-in/verify`)

- 6-digit input boxes (single combined input that auto-advances visually).
- "Resend code" link (60s cooldown shown as countdown).
- Wrong code → inline error. After 5 wrong attempts → 60s lockout (Clerk default).
- Success → `POST /v1/onboarding/start` → redirect to `next_step`.

### 4. Language confirmation (`/[locale]/onboarding/language`)

- Pre-selected based on welcome screen choice.
- Question: "Is this your preferred language?" / "¿Es este tu idioma preferido?"
- Toggle + Continue.

### 5. Resume upload (`/[locale]/onboarding/resume`)

- Two large buttons:
    - **"I have a resume"** → file picker (`accept="application/pdf,application/vnd.openxmlformats-officedocument.wordprocessingml.document"`).
    - **"I don't have one yet"** → skip directly to profile editor with empty form.
- Drag-and-drop on desktop; tap-to-upload on mobile.
- Show selected filename + size + "Upload" button.
- File-size precheck client-side; rejects > 10 MB before uploading.

### 6. Parsing (`/[locale]/onboarding/resume/parsing`)

- Full-screen spinner with rotating helpful messages cycling every 3s:
    - "Reading your resume…"
    - "Pulling out your skills…"
    - "Almost done…"
- Polls `/resume/status` every 2s. Max 60s before showing "Taking longer than expected — keep waiting / start manually" two-button choice.
- Closing the tab: parse continues server-side; user resumes on next visit.

### 7. Profile review/edit (`/[locale]/onboarding/profile`)

- Top: contact card (`firstName`, `lastName`, `phone` from Clerk).
- Sections, each collapsible: Experience, Education, Skills, Certifications, Languages.
- Each section item is editable inline (tap → text input or picker).
- Add/remove items per section.
- Autosave on field blur via `PATCH /v1/onboarding/profile`. Status pill shows "Saving…" / "Saved".
- "Continue" disabled until `firstName + lastName` present.
- If parser was skipped or failed: empty editor with same shape; user fills in manually.

### 8. County (`/[locale]/onboarding/county`)

- 5 large buttons in a 2-column grid: Fresno, Kern, Kings, Madera, Tulare.
- Single-select. Continue enabled on selection.
- Subtitle: "Your county helps us match you to nearby jobs." / "Tu condado nos ayuda a encontrar trabajos cercanos."
- "I'm in another county" link → `/[locale]/onboarding/waitlist`.

### 9. Skills (`/[locale]/onboarding/skills`)

- Multi-select chips. Pre-populated from `resume.skills` if present (limited to known tags; unknown ones become "custom" chips already-selected).
- Default skill list (bilingual) — see [05-i18n.md](05-i18n.md) for keys: Harvesting, Pruning, Irrigation, Tractor Operation, Forklift, Crew Lead, Bilingual Communication, Packing, Equipment Repair, Pesticide Application.
- Custom add: "+ Add skill" → text input (saved as a non-default tag).
- Min 1, max 20. Counter visible.

### 10. Availability (`/[locale]/onboarding/availability`)

- 7-row × 2-column grid (days × AM/PM).
- Tap a cell to toggle.
- "Mark all" / "Clear all" buttons above the grid.
- Optional notes textarea (280 chars, e.g., "no Sundays" / "no domingos").
- Continue disabled until at least one cell is true.

### 11. Complete (`/[locale]/onboarding/complete`)

- Simple checkmark hero (no confetti — keeps bundle small).
- Headline: "You're all set!" / "¡Listo!"
- Up to 5 recommended job cards based on county + skill match (server-side render, not a fetch).
- Two CTAs: "See all jobs" → `/[locale]/jobs` and "View my profile" → `/[locale]/profile`.

### Waitlist (`/[locale]/onboarding/waitlist`)

- Heading: "AgConn serves the Central Valley today" / "AgConn sirve al Valle Central por ahora"
- Form: email or phone (one required) + free-text county.
- Submit → `POST /v1/onboarding/waitlist`.
- After submit: thank-you message with optional EN/ES "follow on social" links (when those exist).

## States (per screen)

| state   | display                                                                                                   |
| ------- | --------------------------------------------------------------------------------------------------------- |
| Loading | Skeleton matching final layout (no layout shift). Avoid spinners on text content.                         |
| Empty   | Per-screen empty-state copy in [05-i18n.md](05-i18n.md).                                                  |
| Error   | Inline error above the primary CTA, with a retry button. Generic 5xx → "Something went wrong. Try again." |
| Offline | Full-screen "You're offline" overlay with cached previous step + auto-retry on reconnect.                 |

## Accessibility

- All form inputs have visible `<label>` and `aria-describedby` for errors.
- Tap targets ≥ 44×44 CSS px (Apple guideline) — daisyUI's default `btn-lg` meets this.
- Color contrast ≥ 4.5:1. The only safe pair is `text-base-content` on `bg-base-100` from daisyUI; verify in Lighthouse audit.
- Screen reader: each step's progress announced via `aria-live="polite"` ("Step 4 of 8 — County").
- Keyboard navigation: Tab order matches visual order. Enter advances within forms.
- Reduced motion: respect `prefers-reduced-motion` for any spinner / transition.

## Mobile / PWA

- Designed mobile-first; desktop is just larger spacing (max-width 480px content column).
- Service worker caches the onboarding shell so users can resume after tab kill.
- IndexedDB stores in-progress profile JSON keyed by `userId`; flushes to API on reconnect via background sync.
- `manifest.json` has bilingual `name` / `short_name`; install prompt shown after step 3.
