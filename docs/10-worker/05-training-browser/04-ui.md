# 05 — Training Browser: UI

## Routes

| route | screen |
|---|---|
| `/[locale]/training` (or `/es/capacitacion`) | listing + filters |
| `/[locale]/training/[slug]` | program detail |
| `/[locale]/me/training` | worker's own enrollments |
| `/[locale]/org/training` | training-org dashboard (auth, role gated) |
| `/[locale]/org/training/new` | create program |
| `/[locale]/org/training/:id` | edit program + roster |

## Worker listing

Similar to job listing pattern. Cards show:

- Title (active locale)
- Funder badge (CDFA / F3 / CalOSBA / EDD / Other)
- County
- Start date – end date
- Spots left ("12 of 20 spots left" / "12 de 20 espacios disponibles")
- "Funded — free" pill (training programs are typically free for workers)

Filter sheet: county, funder, topic chips, date range, "show only programs with capacity" toggle.

## Worker enrollments

`/me/training` similar to `/applications`:

- Tabs: Upcoming / Completed / Dropped
- Each card shows: program title, dates, status, certificate link if completed
- Tap card → enrollment detail with full session schedule and unenroll option (if upcoming)

### Roster view

```
┌────────────────────────────────┐
│  Tractor Safety — Roster       │
│  20 enrolled                   │
│                                │
│  [Mark completed: Maria L. ✓]  │
│  [Mark dropped: Juan H.]       │
│  [No-show: Pedro M.]           │
│  ...                           │
└────────────────────────────────┘
```

Bulk-mark options: "Mark all completed" with confirmation, "Mark no-show for non-completed".

> **Inferred:** Bulk mark is essential — orgs run cohorts of 20+ workers. Marking each individually is too slow. Bulk operation is the default; per-worker override after.

## States

- **Empty listing:** "No training programs scheduled right now. Check back soon — we'll text you if you saved a search." (post-MVP wait-list)
- **Empty enrollments:** "You haven't enrolled in any training yet. [Browse training]"
- **Org with no programs:** wizard-like "Let's create your first program" CTA.

## Public SEO

`/training/[slug]` is a public, indexable page with:

- `EducationalOccupationalProgram` JSON-LD
- Direct-answer paragraph
- All bilingual content visible (no client-only rendering)

## Components

- `<ProgramCard>`, `<ProgramDetail>`, `<ProgramFilters>`, `<EnrollmentCard>`, `<RosterTable>`, `<MarkCompleteModal>` — in `apps/web/app/[locale]/training/_components/` and `_org/`.
