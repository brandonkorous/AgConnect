# 11 — App Shell: i18n

All copy emitted by app-shell primitives lives under the `shell.*` namespace in `packages/i18n/messages/{en,es}.json`. This is the only namespace that `packages/ui` and `packages/api-client` are allowed to read.

Every key below has both `en` and `es` values. The CI parity check (see [04-i18n](../04-i18n/)) fails the build if any key is missing in either locale.

## Namespace

```
shell.
  error.<code>.title           # one entry per error code in 03-api.md
  error.<code>.description     # optional supporting line
  validation.<zod_code>        # zod issue.code → message
  toast.<role>                 # aria-label for toast region
  modal.confirm
  modal.cancel
  modal.close
  form.required_indicator
  form.submit_default
  form.submitting
  page_error.404.title
  page_error.404.description
  page_error.500.title
  page_error.500.description
  page_error.try_again
  page_error.go_home
  page_error.error_id
  install.title
  install.body
  install.cta
  install.later
  install.ios.title
  install.ios.body

  install.ios.add_step
  offline.banner

```

## Reviewer guidance

Spanish copy in this namespace follows [docs/brand/04-voice.md](../../brand/04-voice.md):

- **Tuteo (`tú`), not `usted`.** AGCONN speaks as a peer.
- **Plain present-tense verbs.** Avoid future-conditional pile-ups (`hubieras tenido que…`).
- **Short sentences.** Match the EN reading level.
- **No anglicisms.** "Inicia sesión" not "Loguéate". "Recarga" not "Refresca". "Trabajo" not "Job".
- **Inclusive but readable.** Avoid `@`/`x` morphology in shell strings; use plurals or neutral phrasings (`Tu cuenta`, `Bienvenido a AGCONN`).

The native-Spanish reviewer (per Phase A definition of done) signs off on shell strings before any feature ships.

## CI parity check

`packages/i18n/scripts/check-parity.ts` walks both locale files and asserts:

1. Every key present in `en.json` is present in `es.json` and vice versa.
2. Every value is a non-empty string (empty strings are allowed only for known-empty entries — `error.aborted.title`, `error.aborted.description`, descriptions deliberately blank).
3. Every interpolation token (`{name}`, `{count}`) appears in both translations of the same key.

Run in CI; fail the build on mismatch.

## Adding a new error code

When a new error code is added in [03-api.md](03-api.md):

1. Add `shell.error.<new_code>.title` + `shell.error.<new_code>.description` to both locale files.
2. Add an entry to the standard error codes table in `03-api.md`.
3. If the code should auto-toast, set the default `toast` value in the table.
4. CI fails until both locale entries exist.
