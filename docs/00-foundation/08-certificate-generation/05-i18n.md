# 08 — Certificate Generation: i18n

The certificate PDF itself is bilingual — every label appears in EN and ES on the same page. There's no per-locale variant; the document serves both languages simultaneously.

## Static labels (in the PDF)

| EN | ES |
|---|---|
| Certificate of Completion | Certificado de Finalización |
| This certifies that | Esto certifica que |
| has successfully completed | ha completado con éxito |
| Certificate ID | ID de certificado |
| Issued by | Emitido por |
| Authorized Signatory | Firmante Autorizado |
| Funded by | Financiado por |

These live in `packages/pdf/src/templates/Certificate.strings.ts`:

```ts
export const certStrings = {
  certificate_of_completion: { en: 'Certificate of Completion', es: 'Certificado de Finalización' },
  this_certifies_that:       { en: 'This certifies that', es: 'Esto certifica que' },
  has_completed:             { en: 'has successfully completed', es: 'ha completado con éxito' },
  certificate_id:            { en: 'Certificate ID', es: 'ID de certificado' },
  issued_by:                 { en: 'Issued by', es: 'Emitido por' },
  authorized_signatory:      { en: 'Authorized Signatory', es: 'Firmante Autorizado' },
  funded_by:                 { en: 'Funded by', es: 'Financiado por' },
};
```

## Dynamic content (from DB)

- **Worker name:** rendered as-is from `worker_profiles.first_name + last_name`. No translation.
- **Program title:** rendered as both `program.titleEn` AND `program.titleEs` on separate lines.
- **Completion date:** formatted in both EN and ES using `Intl.DateTimeFormat`:
  - EN: "January 15, 2026"
  - ES: "15 de enero de 2026"
- **Signature name + title + business name:** rendered as-is. No translation.
- **Funder name:** rendered as-is (e.g., "CDFA", "F3"). These are abbreviations / proper nouns; no translation.

## Date formatting helpers

```ts
// packages/pdf/src/format.ts
export const fmtDateEn = (d: Date) =>
  new Intl.DateTimeFormat('en-US', { dateStyle: 'long' }).format(d);
export const fmtDateEs = (d: Date) =>
  new Intl.DateTimeFormat('es-MX', { dateStyle: 'long' }).format(d);
```

## Layout decisions for bilingual content

- Headers stack EN then ES (EN on top, ES smaller below).
- Worker name: single line, no language variation needed.
- Program name: two lines, EN first, ES second, both same size.
- Dates: single line `<EN date> / <ES date>`.

> **Inferred:** EN-on-top reflects formality conventions of US institutional documents while still giving Spanish equivalent prominence. Some grant funders may prefer ES-first; if so, swap order via a per-tenant setting (out of scope for MVP).

## Font choice

Inter (Google Fonts) — clean, supports all Spanish accent characters, free for embedding. Subset to Latin + Latin Extended-A to keep PDF size small (~150 KB total).

## CI checks

- `check-cert-strings-parity` — every key in `certStrings` has both `en` and `es`.
- Snapshot test renders a sample certificate PDF and compares against a checked-in fixture; layout regressions fail CI.
